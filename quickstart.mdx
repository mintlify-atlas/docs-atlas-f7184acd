Get stereOS running in minutes with this quickstart guide. You'll build a development image, launch a VM, and connect to it via SSH.

## Prerequisites

- Linux or macOS system
- [Nix](https://nixos.org/download.html) installed with flakes enabled
- QEMU (installed automatically via Nix dev shell)
- 8GB RAM and 20GB disk space

## Quick start

<Steps>
  <Step title="Clone the repository">
    ```bash
    git clone https://github.com/papercomputeco/stereOS.git
    cd stereOS
    ```
  </Step>

  <Step title="Enter the development shell">
    The dev shell provides all build tools automatically:
    
    ```bash
    nix develop --impure
    ```
    
    <Note>
    If you have direnv installed, the shell will activate automatically when you cd into the directory.
    </Note>
  </Step>

  <Step title="Build the development image">
    Build the OpenCode mixtape with SSH key injection:
    
    ```bash
    make build-qcow2 MIXTAPE=opencode-mixtape-dev
    ```
    
    This creates `result/stereos.qcow2` (approximately 2-4GB). First build takes 10-15 minutes.
  </Step>

  <Step title="Build kernel artifacts">
    For fast boot, build the kernel artifacts:
    
    ```bash
    make build-kernel MIXTAPE=opencode-mixtape-dev
    ```
    
    This creates `result-kernel/` with bzImage, initrd, and cmdline.
  </Step>

  <Step title="Launch the VM">
    ```bash
    make run
    ```
    
    The VM boots in 2-3 seconds using direct kernel boot. You'll see the stereOS ASCII art banner:
    
    ```
     ______   ______  ______   ______   ______   ______   ______
    /\  ___\ /\__  _\/\  ___\ /\  == \ /\  ___\ /\  __ \ /\  ___\
    \ \___  \\/_/\ \/\ \  __\ \ \  __< \ \  __\ \ \ \/\ \\ \___  \
     \/\_____\  \ \_\ \ \_____\\ \_\ \_\\ \_____\\ \_____\\/\_____\
      \/_____/   \/_/  \/_____/ \/_/ /_/ \/_____/ \/_____/ \/_____/
    
        Mixtape: opencode-mixtape
    ```
  </Step>

  <Step title="SSH into the VM">
    In a new terminal, connect as the admin user:
    
    ```bash
    make ssh-admin
    ```
    
    Or connect as the agent user:
    
    ```bash
    make ssh-agent
    ```
    
    <Info>
    The dev build automatically injects your SSH key from `~/.config/stereos/ssh-key.pub`.
    </Info>
  </Step>

  <Step title="Run an AI agent">
    Once connected as the agent user, run OpenCode:
    
    ```bash
    # Set your API key
    export ANTHROPIC_API_KEY="your-key-here"
    
    # Run OpenCode
    opencode
    ```
    
    The agent runs in a restricted environment with limited package access and no Nix daemon access.
  </Step>
</Steps>

## What you built

The development build includes:

- **OpenCode agent** - AI coding assistant binary
- **SSH key injection** - Your SSH key from `~/.config/stereos/ssh-key.pub`
- **Two users**:
  - `admin` - Full system access with sudo
  - `agent` - Restricted shell, limited packages, no Nix access
- **Two daemons**:
  - `stereosd` - System lifecycle and control plane
  - `agentd` - Agent process management

## Common make targets

| Target | Description |
|--------|-------------|
| `make build` | Build raw EFI image |
| `make build-qcow2` | Build QCOW2 image |
| `make build-kernel` | Build kernel artifacts |
| `make run` | Launch VM with QEMU |
| `make ssh-admin` | SSH as admin user |
| `make ssh-agent` | SSH as agent user |
| `make dist` | Build all formats with compression |

## Environment variables

Customize builds with environment variables:

```bash
# Build a different mixtape
make build MIXTAPE=claude-code-mixtape-dev

# Change architecture
make build ARCH=x86_64-linux

# Use a different SSH port
make run SSH_PORT=3333
```
