The `mkDist` function assembles a complete distribution directory from pre-built stereOS image artifacts, compressing them and generating checksums for downstream consumers.

## Function Signature

```nix
mkDist :: {
  pkgs     :: Nixpkgs package set
  name     :: string
  version  :: string (default: "0.0.0-dev")
  system   :: string (default: "aarch64-linux")
  raw      :: derivation
  qcow2    :: derivation
  kernel   :: derivation
} -> derivation
```

## Returns

A derivation containing:

- `stereos.img` - Raw EFI disk image (uncompressed)
- `stereos.img.zst` - zstd-compressed raw image
- `stereos.qcow2` - QCOW2 disk image (uncompressed)
- `stereos.qcow2.zst` - zstd-compressed QCOW2 image
- `bzImage` - Compressed kernel
- `initrd` - Initramfs
- `cmdline` - Kernel command line
- `init` - NixOS stage-2 init path
- `mixtape.toml` - Build manifest with SHA-256 checksums and sizes for all artifacts

## Usage

### Basic Usage

```nix flake/images.nix
{ self, inputs, ... }:

let
  dist-lib = import ../lib/dist.nix { inherit inputs; };
in
{
  perSystem = { pkgs, system, ... }: {
    packages = {
      opencode-mixtape-dist = dist-lib.mkDist {
        inherit pkgs system;
        name = "opencode-mixtape";
        version = "0.1.0";
        raw = self.nixosConfigurations.opencode-mixtape.config.system.build.raw;
        qcow2 = self.nixosConfigurations.opencode-mixtape.config.system.build.qcow2;
        kernel = self.nixosConfigurations.opencode-mixtape.config.system.build.kernel-artifacts;
      };
    };
  };
}
```

## Compression

Images are compressed using zstd with maximum compression:

```bash
zstd -19 -T0 stereos.img -o stereos.img.zst
```

- `-19` - Maximum compression ratio (trades CPU time for smaller size)
- `-T0` - Use all available CPU threads

Typical compression ratios:
- Raw image: 2.1 GB → 512 MB (4:1)
- QCOW2 image: 1.8 GB → 450 MB (4:1)

## Checksums

All checksums are SHA-256, computed at build time:

```bash
sha256sum stereos.img | cut -d' ' -f1
```

This ensures downstream consumers (Dagger, CI) never need to re-hash multi-GB files.

## Use Cases

### CI/CD Pipeline

```yaml .github/workflows/release.yml
- name: Build distribution
  run: nix build .#opencode-mixtape-dist

- name: Parse manifest
  run: |
    VERSION=$(grep '^version' result/mixtape.toml | cut -d'"' -f2)
    echo "MIXTAPE_VERSION=$VERSION" >> $GITHUB_ENV

- name: Upload release artifacts
  uses: actions/upload-artifact@v3
  with:
    name: opencode-mixtape-${{ env.MIXTAPE_VERSION }}
    path: result/*
```

```nix
mkDist = { pkgs, name, version ? "0.0.0-dev", system ? "aarch64-linux", raw, qcow2, kernel }:
  pkgs.runCommand "${name}-dist" {
    inherit version;
    mixtape = name;
    arch = system;
    nativeBuildInputs = [ pkgs.zstd ];
  } ''
    mkdir -p $out
    
    # Copy artifacts
    cp -L ${raw}/stereos.img      $out/
    cp -L ${qcow2}/stereos.qcow2  $out/
    cp -L ${kernel}/bzImage        $out/
    cp -L ${kernel}/initrd         $out/
    cp -L ${kernel}/cmdline        $out/
    cp -L ${kernel}/init           $out/
    
    # Compress with zstd
    zstd -19 -T0 $out/stereos.img   -o $out/stereos.img.zst
    zstd -19 -T0 $out/stereos.qcow2 -o $out/stereos.qcow2.zst
    
    # Compute checksums and generate mixtape.toml
    # ...
  '';
```
